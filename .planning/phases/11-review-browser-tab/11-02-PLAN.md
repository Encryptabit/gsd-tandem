**1. SSE Singleton Store (sse-store.ts):**

Per the user's locked decision: "anything SSE related should have a singleton alive for the lifetime of the broker (or page) and features subscribe to an observable or store to get updates instead of teardown and reinit."

Create `sse-store.ts` that wraps the existing `window.gsdSSE` singleton with a reactive store pattern. This does NOT replace sse.ts — it builds on top of it. Existing tabs continue using `window.gsdSSE.subscribe()` directly (deferred: SSE migration for existing tabs).

```typescript
// sse-store.ts — Observable store layer over window.gsdSSE singleton

type StoreCallback<T> = (value: T) => void;

interface Store<T> {
  get(): T;
  subscribe(callback: StoreCallback<T>): () => void;  // returns unsubscribe fn
}

function createSSEStore<T>(eventType: string, initialValue: T, transform?: (raw: unknown) => T): Store<T> {
  let current: T = initialValue;
  const listeners = new Set<StoreCallback<T>>();

  // Subscribe to SSE events via the existing window.gsdSSE singleton
  function handleEvent(data: unknown): void {
    current = transform ? transform(data) : data as T;
    listeners.forEach(cb => { try { cb(current); } catch(e) { console.error('[SSE Store]', e); } });
  }

  // Wire up when gsdSSE is available
  function connect(): void {
    if (window.gsdSSE) {
      window.gsdSSE.subscribe(eventType, handleEvent);
    } else {
      // Retry after scripts load
      setTimeout(connect, 100);
    }
  }
  connect();

  return {
    get() { return current; },
    subscribe(callback: StoreCallback<T>) {
      listeners.add(callback);
      // Immediately emit current value to new subscriber
      if (current !== initialValue) {
        try { callback(current); } catch(e) { console.error('[SSE Store]', e); }
      }
      return () => { listeners.delete(callback); };
    }
  };
}

export { createSSEStore, Store, StoreCallback };
```

Export `createSSEStore` and the `Store` type for use by reviews.ts and future tab scripts.

**State management:**
- `let currentReviews: ReviewListItem[] = []` — current list data
- `let currentFilters = { status: '', category: '', sort: 'updated_desc' }` — active filters
- `let savedScrollPosition = 0` — scroll position preservation (locked decision)
- `let currentDetailId: string | null = null` — which review is shown in detail view (null = list view)

**Functions:**

`fetchReviews()`: Build URL from currentFilters, fetch from `/dashboard/api/reviews?status=X&category=X&sort=X`. Parse JSON, update currentReviews, call renderList().

`renderList()`: Clear `review-cards` container. If no reviews, show `review-empty`, hide cards container. Otherwise, hide empty state, iterate currentReviews and create card elements using `createCardElement()`. Use DocumentFragment for batch insertion (same pattern as logs.ts). Update `review-count` label with total.

`createCardElement(review: ReviewListItem)`: Create a div.review-card element. Structure:
- Header row: status badge (colored span) + intent text (truncated to ~120 chars) + message count badge if >0
- Meta row: category label, priority badge (if not normal), agent_type, phase/plan/task, timestamps
- Footer row: verdict_reason (truncated) if present, relative time since updated_at

Use `escapeHtml()` helper (same pattern as overview.ts). Status badge colors via CSS class: `.review-status-pending`, `.review-status-approved`, etc. Priority badge: `.review-priority-critical` (red), `.review-priority-low` (muted).

Format timestamps as relative time: "2m ago", "1h ago", "3d ago" using a `formatRelativeTime(isoString)` helper.

Add click handler on each card: calls `showDetail(review.id)`.

`showDetail(reviewId: string)`:
- Save scroll position: `savedScrollPosition = document.getElementById('review-list-view')?.scrollTop || window.scrollY`
- Set `currentDetailId = reviewId`
- Hide list view (`review-list-view` display none), show detail view (`review-detail-view` display block)
- Set detail title from review intent
- Dispatch a custom event `review-detail-open` with the reviewId for Plan 03 to handle detail tab rendering

`showList()`:
- Set `currentDetailId = null`
- Hide detail view, show list view
- Restore scroll position from `savedScrollPosition`

**Filter/sort event handlers:**
Wire up change events on the three dropdowns (`review-filter-status`, `review-filter-category`, `review-sort`). On change, update `currentFilters` and call `fetchReviews()`.
