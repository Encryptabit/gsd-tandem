**Diff tab rendering (with syntax highlighting):**

`renderDiff(diffText: string | null, container: HTMLElement)`: Populate `review-tab-diff`.

If diff is null or empty, show "No diff available" message.

Otherwise, parse the unified diff text into file sections:

`parseDiff(diffText: string)`: Parse unified diff format into structured data:
```typescript
interface DiffFile {
  header: string;        // "diff --git a/file b/file" or "--- a/file"
  oldFile: string;       // filename from ---
  newFile: string;       // filename from +++
  hunks: DiffHunk[];
}
interface DiffHunk {
  header: string;        // "@@ -1,3 +1,4 @@ context"
  lines: DiffLine[];
}
interface DiffLine {
  type: 'add' | 'remove' | 'context' | 'header';
  content: string;
  oldLineNum: number | null;
  newLineNum: number | null;
}
```

Split on `diff --git` boundaries (or `---` if no git prefix). Parse hunk headers (`@@`). Classify lines by first character: `+` = add, `-` = remove, ` ` = context.

Track line numbers: context and remove lines increment old line number, context and add lines increment new line number.

**Syntax highlighting (self-contained, no CDN):**

Build a lightweight, self-contained syntax highlighter. Do NOT use any external library or CDN (per user decision). Implement keyword-based highlighting for common languages:

`highlightLine(content: string, language: string)`: Apply token-based coloring.

Language detection from file extension in the diff header:
- `.ts`, `.tsx`, `.js`, `.jsx` → JavaScript/TypeScript
- `.py` → Python
- `.md` → Markdown
- `.css` → CSS
- `.html`, `.astro` → HTML
- `.json` → JSON
- `.sql` → SQL
- `.yaml`, `.yml` → YAML
- Default → plain text (no highlighting)

For each language, define simple regex-based token patterns:
- **JS/TS**: keywords (`const`, `let`, `var`, `function`, `async`, `await`, `return`, `import`, `export`, `if`, `else`, `for`, `while`, `class`, `interface`, `type`, `new`, `throw`, `try`, `catch`, `finally`, `switch`, `case`, `break`, `default`, `true`, `false`, `null`, `undefined`), strings (single/double/backtick quoted), comments (`//` to end of line), numbers, template literal expressions
- **Python**: keywords (`def`, `class`, `import`, `from`, `return`, `if`, `elif`, `else`, `for`, `while`, `try`, `except`, `finally`, `with`, `as`, `yield`, `async`, `await`, `True`, `False`, `None`, `and`, `or`, `not`, `in`, `is`, `lambda`, `pass`, `raise`, `break`, `continue`), strings (single/double/triple quoted), comments (`#` to end of line), numbers, decorators (`@`)
- **CSS**: properties, values, selectors, comments
- **HTML**: tags, attributes, strings
- **JSON**: keys, strings, numbers, booleans, null

Wrap highlighted tokens in spans with CSS classes: `.hl-keyword`, `.hl-string`, `.hl-comment`, `.hl-number`, `.hl-type`, `.hl-decorator`, `.hl-tag`, `.hl-attr`, `.hl-punctuation`.

All content MUST be HTML-escaped FIRST (escapeHtml) before regex highlighting is applied, to prevent XSS. Then wrap matched tokens in spans.
